# Version Control Portal 🚀

Добро пожаловать в **Version Control Portal** — современную платформу для контроля версий и конфигураций, разработанную для эффективной работы в команде. 

Эта система помогает разработчикам и менеджерам организовывать проекты, управлять версиями кода, отслеживать изменения и обеспечивать безопасный совместный доступ к ресурсам.

## 🎯 Ключевые возможности

*   **Управление проектами и версиями:** Удобное создание проектов, загрузка файлов и прозрачная история версий.
*   **Ролевая модель:** Динамическое разграничение прав доступа (Admin, Developer, Viewer). У каждого пользователя есть свой профиль и аватар.
*   **Контроль доступа к проектам:** Изолированные рабочие пространства. Разработчики и зрители могут просматривать и работать только с теми проектами, куда их добавил администратор. Добавление происходит через удобные модальные окна со списком выбора.
*   **Безопасность:** Надежная авторизация и защита данных, безопасное хранение паролей.
*   **Современный интерфейс:** Быстрый, интуитивно понятный и отзывчивый интерфейс с поддержкой закрепленных вкладок и загрузки медиа-файлов.

## 🔐 Контроль доступа (Пользователи и Проекты)

В системе реализован гибкий механизм управления видимостью проектов на уровне отдельных участников:
*   **Изоляция данных:** Обычные разработчики и зрители видят только те проекты, в которые они были добавлены. Это позволяет держать разные команды в единой системе без пересечения данных.
*   **Удобное управление:** Назначать права можно с двух сторон — добавить пользователей в конкретный проект (вкладка «Участники» на странице проекта) или добавить проекты конкретному пользователю (прямо в его профиле).
*   **Массовое выделение:** Специальные интерфейсы с таблицами и чекбоксами позволяют в пару кликов добавить сразу десяток участников в проект.
*   **Автоматический учет авторов:** Если пользователь создал хотя бы одну версию в проекте, система автоматически фиксирует его как участника данного проекта.
*   **Абсолютные права администратора:** Данные ограничения не распространяются на пользователей с ролью **Admin**. Администраторы по умолчанию видят, просматривают и могут изменять абсолютно любые проекты в системе. Им также доступен эксклюзивный функционал управления доступом других пользователей.

## 🛠 Общий стек технологий

*   **Frontend:** React, TypeScript, Vite, Ant Design (UI).
*   **Backend:** Node.js, NestJS, Prisma ORM.
*   **База данных:** PostgreSQL (разворачивается через Docker).

## 🚀 Быстрый старт (Запуск всего проекта)

Проект настроен так, чтобы вам не приходилось запускать каждую часть (фронтенд и бэкенд) отдельно. Всё управление вынесено в корень проекта.

### Шаг 1. Предварительная настройка
1. Убедитесь, что у вас установлен **Node.js** и **Docker Desktop**.
2. Запустите Docker Desktop в фоновом режиме.
3. В папке `backend` создайте файл `.env` (можете скопировать содержимое из `.env.example`).

### Шаг 2. Установка и запуск

Откройте терминал в корневой папке проекта (`version-control-portal`) и выполните всего две команды:

1. **Установите все зависимости разом** (в корне, фронтенде и бэкенде):
   ```bash
   npm run install:all
   ```

2. **Запустите проект:**
   ```bash
   npm run all:dev
   ```

Эта команда автоматически поднимет базу данных в Docker, проведет миграции, заполнит начальные данные (тестовые проекты и пользователи) и одновременно запустит серверы фронтенда и бэкенда в режиме горячей перезагрузки.

🌐 Приложение будет доступно по адресу: **http://localhost:5173**
*(Данные для входа тестового администратора: логин `admin`, пароль `admin123`)*
