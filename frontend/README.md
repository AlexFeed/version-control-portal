# 🖥️ Frontend — Version Control Portal

Клиентская часть системы контроля версий и конфигураций. Разработана на **React** с использованием сборщика **Vite**. В качестве UI-библиотеки применяется **Ant Design (antd)**. 

Проект полностью интегрирован с реальным бэкендом (NestJS), использует JWT-авторизацию и работает с реальной базой данных.

---

## 🚀 Быстрый запуск

1. Перейдите в директорию \`frontend\` и установите зависимости:
   ```bash
   npm install
   ```
2. Запустите проект:
   ```bash
   npm run dev
   ```

Проект будет доступен по адресу \`http://localhost:5173\`.

---

## 🛠 Основные особенности и стек

* **Стек:** React 18, TypeScript, Vite.
* **UI-кит:** Ant Design — предоставляет готовые компоненты (таблицы, модальные окна, формы, бейджи, графики).
* **Стилизация:** Чистый CSS с использованием CSS-переменных для поддержки светлой/темной темы (через ConfigProvider antd).
* **Роутинг:** React Router DOM (v6).
* **Авторизация:** JWT-токены. Токен сохраняется в `localStorage`, к каждому запросу автоматически прикрепляется заголовок `Authorization: Bearer <token>`.
* **Работа с сетью:** Настроен кастомный `fetchApi` с автоматической обработкой ошибок и перенаправлением на страницу логина при истечении токена (401 Unauthorized).

## 🗂 Структура проекта

* `src/api` — функции для общения с бэкендом (пользователи, проекты, версии).
* `src/auth` — контекст авторизации (AuthContext) и провайдер (определяет текущего пользователя и его роль).
* `src/components` — переиспользуемые UI-компоненты (Layout с навигацией, форматирование дат, теги).
* `src/pages` — основные экраны приложения (Логин, Дашборд, Проекты, Участники).
* `src/pinned` — функционал закрепленных вкладок (доступен на всех страницах).

## 🔒 Доступ и роли

Интерфейс динамически адаптируется под роль пользователя (Admin, Developer, Viewer):
* **Viewer** может только просматривать проекты и изменения.
* **Developer** может создавать новые проекты, управлять их версиями.
* **Admin** обладает полными правами, включая возможность создания новых пользователей, их удаления и управления доступом.

## 🖼️ Загрузка файлов (Аватары)

Во фронтенде реализована загрузка изображений профиля через компонент `Upload` от antd. Картинки загружаются на бэкенд (сохраняются на диске) и раздаются фронтенду через статический сервер Vite-прокси, предотвращая проблемы с CORS и путями.
